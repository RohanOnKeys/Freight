# Infrastructure

## Configuration

```
freight/core/
```

### config.py

Responsible for loading and validating every environment variable.

Provides a single global `settings` object used throughout the application.

---

## Database

```
freight/db/
```

### base.py

Contains the application's declarative SQLAlchemy `Base`.

Every model inherits from this class.

### session.py

Creates

* SQLAlchemy Engine
* Session Factory
* `get_db()` dependency

Responsible for database connection management.

# Database Models

```
freight/models/
```

## Pipeline

Represents one CI/CD execution.

One GitHub push creates one pipeline.

```
Pipeline

id
repo
commit_sha
branch
status
created_at
```

Relationship

```
Pipeline (1)

        │

        ▼

Jobs (Many)
```

---

## Runner

Represents a worker capable of executing jobs.

```
Runner

id
hostname
status
last_heartbeat
```

Relationship

```
Runner (1)

        │

        ▼

Jobs (Many)
```

---

## Job

Core execution model.

Every stage inside a pipeline becomes one Job.

```
Job

id
pipeline_id
runner_id
name
stage
needs
image
script
status
retries
exit_code
started_at
finished_at
```

Relationships

```
Pipeline

     │

     ▼

    Job

     ▲

     │

Runner
```

---

## Artifact

Represents files generated by completed jobs.

```
Artifact

id
job_id
path
size
created_at
```

Relationship

```
Job (1)

     │

     ▼

Artifacts (Many)
```

---

## Secret

Stores encrypted credentials.

Secrets are decrypted only during job execution.

```
Secret

id
name
encrypted_value
scope
```
